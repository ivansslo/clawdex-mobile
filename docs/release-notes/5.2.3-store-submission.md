3. Open the Git screen to view status/diff.
4. If prompted, review and approve an action.

New in 5.2.3 — Push notifications:
- The bridge sends a push when an agent turn completes or needs approval (the
  app's socket closes when backgrounded, so the bridge is the sender). Delivery

- To exercise it: enable notifications when prompted (or in Settings), background
  the app, then trigger a turn on the review bridge. A notification arrives;
  approval notifications include Approve/Deny actions.

  name, and a short preview (<=140 chars) of the agent's reply. No prompts, code,

## App Privacy (App Store) / Data Safety (Play) — answers

Declare these for 5.2.3 because, when notifications are enabled, a reply snippet
and a push token leave the device via Expo and the platform push service.

Data types collected:
1. Other user content — a truncated (<=140 char) snippet of the assistant's reply,
   included in turn-completed notifications.
   - Linked to the user's identity: No
   - Used for tracking: No
   - Purpose: App Functionality (delivering user-enabled notifications)
   - Optional: Yes (controlled by the in-app notifications toggle)
2. Device ID / push token — an opaque identifier used solely to route the
   notification to the device.
   - Linked to the user's identity: No
   - Used for tracking: No
   - Purpose: App Functionality
   - Optional: Yes

No advertising identifiers. No data sold. No data used for tracking across apps.
